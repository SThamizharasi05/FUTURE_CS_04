# API Security Risk Analysis

## Overview
This project analyzes security risks in APIs and identifies vulnerabilities such as insecure endpoints, weak authentication, and data exposure.

## Objectives
- Identify insecure API endpoints
- Analyze authentication and authorization
- Detect input validation issues
- Check rate limiting

## Tools Used
- Postman / Insomnia
- Browser Developer Tools
- Sample Public APIs

## Key Risks
- Insecure endpoints
- Data exposure
- Weak authentication
- Missing rate limiting
- Poor input validation

## Recommendations
- Use strong authentication (JWT / OAuth)
- Implement access control
- Use HTTPS
- Validate inputs
- Apply rate limiting
